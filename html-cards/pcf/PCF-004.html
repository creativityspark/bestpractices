PCF-004 Always handle null or undefined property values in the updateView method

Always handle null or undefined property values in the updateView method. Data may not be ready when updateView is first called, and your component must gracefully handle this state.

1
Guideline 1
Check for null or undefined values on all bound properties before using them.
2
Guideline 2
Provide a visual loading indicator or a sensible default when data is not yet available.
3
Guideline 3
Expect that a subsequent updateView call will include the actual values once data is ready.
1
Rationale 1
The hosting context may call updateView before the bound data has loaded, passing null values for properties that will eventually have data.
2
Rationale 2
Attempting to use null values without checking leads to runtime errors that break the component.
3
Rationale 3
Providing a loading state improves user experience by giving visual feedback that data is being fetched.
✓ Do
  • ✓ public updateView(context: ComponentFramework.Context): void { const value = context.parameters.inputValue.raw; if (value === null || value === undefined) { this._container.innerHTML = ""; this._container.appendChild(this._loadingSpinner); return; } this._renderContent(value); } Language: typescript
✗ Don't
  • ✗ public updateView(context: ComponentFramework.Context): void { // Directly using the value without null checking — will throw if null const value = context.parameters.inputValue.raw; this._container.innerText = value.toString(); } Language: typescript
