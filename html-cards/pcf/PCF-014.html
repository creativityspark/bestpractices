PCF-014 Always provide unit tests with PCF controls

Always provide unit tests with PCF controls. Changes to a control must include automated tests that protect the component against regressions.

1
Guideline 1
Add unit tests for the control's rendering logic, behaviors, validation rules, and output generation.
2
Guideline 2
Run the unit test suite as part of local development and CI before packaging or deploying the control.
3
Guideline 3
Add regression tests whenever fixing a defect so the same issue cannot be reintroduced silently.
4
Guideline 4
Prefer tests that exercise real browser DOM behavior where possible, instead of relying only on shallow mocks.
1
Rationale 1
PCF controls evolve over time, and without unit tests it is easy for small changes to break rendering, validation, or output behavior.
2
Rationale 2
Automated tests give fast feedback and reduce the cost of refactoring or extending existing controls.
3
Rationale 3
Regression tests preserve previously fixed behavior and make code reviews more reliable because reviewers can see that expected scenarios are covered.
