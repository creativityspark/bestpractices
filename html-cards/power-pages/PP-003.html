PP-003 Keep Liquid templates focused on presentation logic

Keep Liquid templates focused on presentation logic. Avoid embedding complex business logic or heavy data processing in Liquid code.

1
Guideline 1
Use Liquid for data shaping and rendering, not for calculations or business rules.
2
Guideline 2
Modularize templates using {% include %} to promote reusability and readability.
3
Guideline 3
Use whitespace control tags ({%- and -%}) to minimize unnecessary whitespace in rendered output.
4
Guideline 4
Always check for null values before accessing object properties to prevent rendering errors.
5
Guideline 5
Escape user-generated content with {{ variable | escape }} to prevent cross-site scripting (XSS).
1
Rationale 1
Liquid is interpreted at runtime on every page load. Complex logic in Liquid increases server-side rendering time and degrades performance.
2
Rationale 2
Business logic belongs in Dataverse plugins or Power Automate flows where it can be tested, versioned, and reused independently.
3
Rationale 3
Modular templates are easier to maintain, debug, and review during code reviews.
4
Rationale 4
Unescaped user content in templates can introduce XSS vulnerabilities.
✓ Do
  • ✓ {%- fetchxml recent_cases -%} {%- endfetchxml -%}
      {% for case in recent_cases.results.entities %}
    • {{ case.title | escape }} - {{ case.createdon | date: "yyyy-MM-dd" }}
      • {% endfor %}
    Language: liquid
✗ Don't
  • ✗ {% assign total = 0 %} {% for item in order_items %} {% assign line_total = item.quantity | times: item.unit_price %} {% assign discount = line_total | times: item.discount_percent | divided_by: 100 %} {% assign total = total | plus: line_total | minus: discount %} {% endfor %}

    {{ user.fullname }}

     Language: liquid
