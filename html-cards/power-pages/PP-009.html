Use content snippets for reusable text, labels, and messages that appear across multiple pages. Avoid duplicating the same content in multiple web templates.

1
Guideline 1
Create content snippets for any text that appears on more than one page (e.g., footer disclaimers, cookie notices, error messages).
2
Guideline 2
Use descriptive names that follow a consistent naming convention (see PPN-001).
3
Guideline 3
Leverage content snippets for multi-language support by providing translated values for each enabled language.
4
Guideline 4
Do not use content snippets for large blocks of HTML or complex Liquid logic; use web templates instead.
1
Rationale 1
Content snippets provide a single source of truth for repeated text, making updates easier and ensuring consistency across the site.
2
Rationale 2
Translators can manage content snippets directly without modifying web templates, simplifying localization workflows.
3
Rationale 3
Duplicating content in multiple templates leads to inconsistencies when one instance is updated but others are forgotten.
✓ Do
  • ✓ A content snippet named VendorName/Footer/Disclaimer containing the legal disclaimer text, referenced in the footer web template with {% snippet 'VendorName/Footer/Disclaimer' %}.
  • ✓ A content snippet named VendorName/Errors/GenericMessage used across all error pages to display a consistent user-friendly error message.
✗ Don't
  • ✗ The same disclaimer text copy-pasted into five different web templates. When the legal team updates the text, only two of the five templates are updated.
  • ✗ A content snippet containing 200 lines of HTML and Liquid code that should be a web template instead.
