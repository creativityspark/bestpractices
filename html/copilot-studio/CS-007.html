Establish a thorough testing strategy and use built-in analytics to continuously improve your agent's performance.

1
Guideline 1
Use the Test bot pane during development to verify every topic's conversation path, including branching and error scenarios.
2
Guideline 2
Create a test script that covers all topics, edge cases, and expected fallback scenarios, and run it before every deployment.
3
Guideline 3
Monitor the Analytics dashboard regularly to track resolution rate, escalation rate, abandonment rate, and customer satisfaction (CSAT).
4
Guideline 4
Review conversation transcripts weekly to identify misunderstood intents, missing topics, and areas for improvement.
5
Guideline 5
Set up alerts or dashboards for critical metrics (e.g., sudden increase in fallback rate or error rate) to catch regressions early.
1
Rationale 1
Untested conversation paths lead to broken experiences that users encounter in production, damaging trust and increasing support costs.
2
Rationale 2
A structured test script ensures consistent coverage and prevents regressions when topics are added or modified.
3
Rationale 3
Analytics provide data-driven insight into how users interact with the agent and where the experience breaks down.
4
Rationale 4
Transcript reviews reveal real user language and uncover gaps that analytics alone cannot surface.
5
Rationale 5
Proactive monitoring catches issues before they affect a large number of users.
✓ Do
  • ✓ A test script documents 50+ test scenarios, including happy paths, edge cases, and fallback triggers, and is executed before every release.
  • ✓ The analytics dashboard is reviewed weekly; a drop in resolution rate from 85% to 70% triggers an investigation and topic updates.
  • ✓ Conversation transcripts reveal that users frequently ask about "PTO balance" but no topic exists - a new topic is created.
✗ Don't
  • ✗ Topics are published without testing; users discover that a critical branch leads to a dead end.
  • ✗ Analytics are never reviewed; the agent's fallback rate has been 40% for months without anyone noticing.
  • ✗ Conversation transcripts are enabled but never read, missing opportunities to improve the agent.
