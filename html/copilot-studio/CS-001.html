CS-001 All topics must use a consistent naming convention that indicates the topic's category...

All topics must use a consistent naming convention that indicates the topic's category and purpose.

1
Guideline 1
Use a Verb-Noun structure for topic names that clearly describes the intent (e.g., Reset Password, Check Order Status).
2
Guideline 2
Use a category prefix to group related topics (e.g., HR - Request Leave, IT - Reset Password, Sales - Get Quote).
3
Guideline 3
Reserve a consistent suffix for sub-topics or dialogs (e.g., - Confirmation, - Error Handling).
4
Guideline 4
Rename all topics immediately after creation - never leave default names like Topic 1 or Untitled.
5
Guideline 5
Keep names concise but descriptive enough to identify the topic's purpose at a glance.
1
Rationale 1
Default names like Topic 1 provide no context and make it difficult to manage agents with many topics.
2
Rationale 2
A Verb-Noun structure makes intent immediately clear when browsing the topic list.
3
Rationale 3
Category prefixes enable quick filtering and grouping in the authoring canvas, especially in agents with dozens of topics.
4
Rationale 4
Consistent naming reduces confusion when multiple makers collaborate on the same agent.
✓ Do
  • ✓ HR - Request Leave - clear category and action
  • ✓ IT - Reset Password - grouped under IT, describes the intent
  • ✓ Order - Check Status - grouped under Orders, specific action
  • ✓ Order - Cancel Order - Confirmation - sub-topic clearly linked to parent
✗ Don't
  • ✗ Topic 1 - default name, no context
  • ✗ Help - too vague, could match anything
  • ✗ Password - no action described, unclear intent
  • ✗ doStuff - meaningless and inconsistent format
