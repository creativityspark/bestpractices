PA-001 Name cloud flows using the standard pattern

Cloud flow names must follow a consistent pattern that identifies the owning app, trigger connector, data source, triggering event, and business purpose.

1
Guideline 1
Use the pattern "[App Prefix] - [Connector Prefix] - [Data Store] - [Event Name Code(s)] - [Action/Purpose]" when naming cloud flows.
2
Guideline 2
Use a 3-letter app prefix to group flows that belong to the same application.
3
Guideline 3
Omit the app prefix when the flow is shared across multiple applications.
4
Guideline 4
Use the connector prefix to identify the connector used by the trigger.
5
Guideline 5
Use the data store segment to identify the table, list, mailbox, app, schedule, or other trigger-backed store.
6
Guideline 6
Omit the data store segment when the trigger does not have an associated data store, such as a manual trigger.
7
Guideline 7
Use event name codes such as NEW, MOD, and DEL, and combine codes when multiple events trigger the flow.
8
Guideline 8
Use a brief action or purpose segment that clearly describes what the flow does.
1
Rationale 1
Having naming conventions in place makes the task of choosing names easier.
2
Rationale 2
The app prefix groups all flows belonging to the same application together in alphabetical lists, making them easier to find and manage.
3
Rationale 3
Including the trigger prefix and data store name makes it easier to identify and select flows in the UI.
4
Rationale 4
Consistent naming conventions make it easier to group similar flows together by name.
5
Rationale 5
The action or purpose segment makes it easy to understand what the flow is doing.
✓ Do
  • ✓ ORM - DV - Contact - NEWMOD - Send Notifications Sends a notification when a Dataverse Contact record is created or modified for the Orders Management app.
  • ✓ ORM - DV - Account - MOD - Validations Performs validations when an Account record is updated in Dataverse for the Orders Management app.
  • ✓ ORM - SP - Orders - MOD - Move out of notice wait period Handles updates to the Orders SharePoint list for the Orders Management app.
  • ✓ HRO - CF - Set Reference Number Names a child flow that sets a reference number for the HR Onboarding app.
  • ✓ HRO - SCH - Daily - Set Status Names a scheduled flow that runs daily and sets status for the HR Onboarding app.
  • ✓ PA - Orders Management - Cancel Record Omits the app prefix because the flow is shared across multiple apps and is triggered from Power Apps.
✗ Don't
  • ✗ Send Notifications Omits the app, connector, data store, and event segments required to make the name self-descriptive.
  • ✗ ORM - Contact - Send Notifications Omits the connector prefix and event code segments.

Pattern: [App Prefix] - [Connector Prefix] - [Data Store] - [Event Name Code(s)] - [Action/Purpose]
